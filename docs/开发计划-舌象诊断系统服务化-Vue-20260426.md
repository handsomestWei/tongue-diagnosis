# 舌象诊断系统 — 开发计划

**文档类型**：开发计划（Implementation Plan）  
**依据设计**：`docs/详细设计-舌象诊断系统服务化-Vue-20260426.md`  
**前端栈**：Vue 3 + Vite + Pinia + Vue Router  
**版本**：v1.8  
**日期**：2026-04-26（**补齐计划「骨架」：批推理后台、增量导出筛选、限流/并发、SAM 契约测、MySQL CI**）  

---

## 1. 计划目标

在约 **5 个阶段（P0–P4）** 内，将现有仓库演进为可交付的 **CPU-only 服务化平台**，包含：

- 后端 API（FastAPI）与可选 Worker；
- Vue 前端（上传、标注、训练、增量训练、推理、纠错）；
- `core/` 核心域分层；
- SQLite / MySQL 可切换存储。  
- **训练与推理均支持两种输入图**：手机自拍「全脸」（须 TongueSAM 抠舌）与「仅舌部特写」（跳过 SAM）；字段 **`image_kind`** 贯穿上传、DB、导出与 API（详设 §5）。
- **Web 账号密码登录 + RBAC**：JWT、路由与 API 双重鉴权（见详设 §7–§8）。

本计划**不写死自然日排期**，以**阶段与任务依赖**为准；每个阶段结束应有**可演示增量**与**最小验收标准**。

---

## 2. 阶段总览

| 阶段 | 名称 | 主要产出 | 验收标准（摘要） |
|------|------|----------|------------------|
| P0 | 骨架与依赖 | 目录、`core` 雏形、CPU 依赖锁定、最小 API | `GET /health`；本地能装依赖无 GPU |
| P1 | 数据层与 MVP API | DB 模型、Alembic、上传/标注 CRUD | SQLite 与 MySQL 各跑通迁移+冒烟 |
| P2 | 推理接入 | InferenceService + 单图/异步批 | 真实权重下 CPU 推理成功（可分步：先仅分类） |
| P3 | 训练与前端 | TrainJob、**按 image_kind 导出规范舌图**、Vue 主要页面 | 浏览器完成「混合上传两种图 → 标注 → 训练 → 推理」且与详设 §5 一致 |
| P4 | 增量与纠错 + 加固 | Correction、增量训练、`predict` 重构 | 纠错写入并可触发增量训练；基础限流/日志 |

## 4. P1 — 数据库、迁移与 MVP 数据 API

### 4.1 任务清单

| ID | 任务 | 说明 |
|----|------|------|
| P1-1 | SQLAlchemy 模型 | User（**username、password_hash、role、is_active**）、Project、Image…… |
| P1-2 | Alembic 初始化 | `migrations/`；`alembic upgrade head` |
| P1-3 | Repository 层 | CRUD 接口，便于单测 |

### 4.2 验收标准

- 切换 `DATABASE_URL` 无需改代码即可完成基本 CRUD。
- OpenAPI 可在线调试上传与标注；**缺 `image_kind` 上传须 400**。

### 4.3 依赖

- P0 完成。

### 4.4 实施备注（截至仓库当前进度）

## 5. P2 — 推理服务（CPU）

### 5.1 任务清单

| ID | 任务 | 说明 |
|----|------|------|
| P2-1 | `core` 中 **`PreprocessForClassify` + `InferenceService`** | 按 **`image_kind`** 分支：`full_face` → TongueSAM+ROI；`tongue_closeup` → 跳过 SAM、与 ROI 后处理对齐的几何/增强 |
| P2-2 | 分类推理 | Ultralytics `YOLO.predict`，`device='cpu'`，**输入为统一规范舌图** |
| P2-3 | TongueSAM 接入 | 仅 `full_face_selfie` 调用；子进程或封装；超时与 **full_face 专用并发** |
| P2-4 | API：`POST /api/v1/infer` | **必传 `image_kind` 或 `image_id`（带出 kind）**；multipart + top-k |
| P2-5 | 批任务表 + `POST /api/v1/infer/batch` | `job_id` + `GET /api/v1/jobs/{id}`；body **含每图 `image_kind` 或 image_id** |
| P2-6 | Worker 最小实现 | 进程内队列或单独脚本消费 Redis（可延后到 P4） |
| P2-7 | 单测 / 契约测 | 同一张图分别用两种 `image_kind` 断言**是否调用 SAM**（mock） |

### 5.2 验收标准

- **各选一张** full_face 与 closeup 样例 + 权重，API 返回结构化结果且路径与详设一致（closeup **无** SAM mock 调用）。
- 并发压测：CPU 下限流不拖垮进程（配置项可调）。

### 5.3 依赖

- P1（需 Image / 存储路径）；权重文件由部署文档约定路径。

### 5.4 实施备注

| 条目 | 当前状态 |
|------|----------|
| `core/preprocess_classify.py` | 按 `image_kind` 分支；特写整图 ROI；全脸依赖外部框 |
| `core/sam_bridge.py` | 子进程 `tongue_sam/predict.py`，读 `test_roi/*.json` |
| `core/inference_yolo.py` | Ultralytics classify top-k（BGR ndarray） |
| `POST /api/v1/infer` | 支持 `file`+`image_kind` 或仅 `image_id`；`CLASSIFY_WEIGHTS_PATH` 空则为演示 |
| `POST /api/v1/infer/batch` + `GET /api/v1/jobs/{id}` | **后台线程**顺序执行 + `async_jobs`（`pending`→`running`→`success`）；替代原先同步长阻塞 |
| 进程内并发 | `INFER_CONCURRENCY` / `INFER_SAM_CONCURRENCY`（`core/infer_slots.py`）；SAM 子进程默认注入 `TONGUESAM_WRITE_VISUAL=0` |
| 限流 | `RATE_LIMIT_PER_MINUTE`（0 关闭）；按 IP / `X-Forwarded-For` |
| SAM 契约 | `tests/test_sam_bridge_contract.py` mock `predict.py` |
| 仍可为后续 | Redis 独立 Worker（多机）、更细粒度路由配额 |

## 6. P3 — 训练流水线与 Vue 前端

### 6.1 后端任务

| ID | 任务 | 说明 |
|----|------|------|
| P3-1 | `TrainJob` 模型与状态机 | pending/running/success/failed |
| P3-2 | 从 DB 导出 YOLO classify 目录 | **对每张图按 `image_kind` 调用 `PreprocessForClassify`**，写出规范舌图至 `train/类名/*`、`val/类名/*`；可写缓存 `derived_tongue_path` |
| P3-2b | `TrainJob.params` | 记录 **各 `image_kind` 样本计数**、imgsz、letterbox/clahe 等与详设对齐 |
| P3-3 | `POST /api/v1/train` | 异步执行；写日志路径 |
| P3-4 | `GET /api/v1/train/{id}` | 状态 + 简要指标 |
| P3-5 | 训练完成后注册 `ModelRegistry` | 路径、指标 JSON |

### 6.2 前端任务（Vue）

| ID | 任务 | 说明 |
|----|------|------|
| P3-F1 | 工程初始化 | Vite + Vue3 + TS（推荐）+ Router + Pinia |
| P3-F2 | HTTP 封装 | axios baseURL `/api`、**Bearer**、401 跳登录 |
| P3-F2b | 角色 UI | 侧栏按角色隐藏；viewer 禁用纠错提交等（后端仍需 enforce） |
| P3-F3 | 页面：上传、标注、训练列表、推理 | 上传 **图类型选择**；标注页显示/修改 **image_kind**；训练、推理与设计「页面信息架构」对应 |
| P3-F4 | 联调 | dev proxy `/api` → 后端 |
| P3-F5 | 生产构建 | `pnpm build`；静态资源由 FastAPI 挂载或 Nginx |

### 6.3 验收标准

- 浏览器完成：**混合上传（全脸 + 特写）→ 标注（含可选改判类型）→ 触发训练 → 查看状态 → 选模型推理**，且训练导出与在线推理共用同一预处理逻辑（抽查日志或单测）。

### 6.4 依赖

- P2 推理链路稳定；P1 数据层稳定。

### 6.5 实施备注（P3 进度）

| 条目 | 状态 |
|------|------|
| `core/train_export.py` | 按 `image_kind` + TongueSAM 导出 `train/val`，写 `derived_tongue_path` |
| `api/train_worker.py` | 后台线程：`export` → `YOLO.train` → `ModelRegistry` |
| `POST/GET /train` | 已跑真实导出+训练（无标注则失败） |
| 推理持久化 | `persist` + `predictions` 表 |
| 纠错 API | `POST /predictions/{id}/correct` |
| 增量导出 | `export_selection` + `merge_base_manual`：`corrections_merged`（默认）= 全量 manual + 勾选纠错覆盖类别；`corrections_only`；`all_manual` |
| Vue | **纠错页每行表单**；**增量训练**页可选择数据策略 |

### 7.2 实施备注（当前）

| 条目 | 说明 |
|------|------|
| TongueSAM `predict.py` | `_ensure_models()` 惰性加载；`if __name__ == "__main__"`；**`TONGUESAM_WRITE_VISUAL`**（API 默认 `0`） |

### 7.3 验收标准

- 用户对错误预测纠错并勾选再训练后，**增量训练任务**能读到该样本。
- 核心接口有基本集成测试（pytest）。
